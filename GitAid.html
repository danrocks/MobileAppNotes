Useful Git knowledge

Mac

Git should be run from the Terminal. Start by navigating to the local git repository folder (make a git folder by importing from remote and running git init).

The terminal can also be used to create files with "touch myNewFile.xxx"

A good synopsis is Using Git for WebSites

To commit and push changes (here):

Add the files to be committed. This stages them for the first commit.
$ git add .

Commit the files that you've staged in your local repository.
$ git commit -m "commit notes"

Push the changes in your local repository to GitHub. $ git push -u origin master

Win 10 - VS2015

VS does make it quite simple to commit changes and push them to github (this blog assumes GitHub has previously been wired up).

Simply navigate to the Team Explorer view and click on the changes button.

Changed files should be listed - eyeball to make sure there are no surprises. Then provide the commit message.

Once message is provided, "Commmit all" is enabled - hit it.

Cloning

I recently had cause to pull files originally created on my MAC into Windows (the files were already in GitHub).

Github's Clone button worked really smoothly, although I did note it chose it's own target directory. I expect there is a way to pick my own directory structure but it isn't immediately obvious.
