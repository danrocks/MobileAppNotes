Useful Git knowledge

Git should be run from the Terminal. Start by navigating to the local git repository folder (make a git folder by importing from remote and running git init).

The terminal can also be used to create files with "touch myNewFile.xxx"

A good synopsis is Using Git for WebSites

To commit and push changes (here):

Add the files to be committed. This stages them for the first commit.
$ git add .

Commit the files that you've staged in your local repository.
$ git commit -m "commit notes"

Push the changes in your local repository to GitHub. $ git push -u origin master
